11 · MOTION

动效

只在状态切换时使用动效，绝不用动画装饰静态元素。缓动一律使用 cubic-bezier(0.16, 0.84, 0.44, 1)（温和减速），避免弹簧式回弹。

时长 Durations

--dur-fast120ms
--dur-base200ms
--dur-slow320ms
默认 200ms。选项切换 / 卡片悬浮用 120ms。题目进出用 320ms。

缓动 Easings

--ease-out
cubic-bezier(0.16, 0.84, 0.44, 1)
--ease-in-out
cubic-bezier(0.4, 0, 0.2, 1)
默认 --ease-out。对话框 / 抽屉进出使用 --ease-in-out。

QuestionCard.vue · 题目卡组件

src/components/QuestionCard.vueVue
<script setup lang="ts">
import { ref } from 'vue';
interface Option { key: string; text: string; }
const props = defineProps<{
  stem: string; options: Option[];
  correctKey: string;
  index: number; total: number;
}>();
const selected = ref<string | null>(null);
const submitted = ref(false);
const stateOf = (k: string) => {
  if (!submitted.value)
    return selected.value === k ? 'selected' : '';
  if (k === props.correctKey) return 'correct';
  if (k === selected.value) return 'wrong';
  return '';
};
const submit = () => { if (selected.value) submitted.value = true; };
</script>

<template>
  <div class="qcard">
    <div class="meta">题号 {{ index }} / {{ total }}</div>
    <div class="stem">{{ stem }}</div>
    <div
      v-for="opt in options" :key="opt.key"
      class="option" :class="stateOf(opt.key)"
      @click="selected = opt.key"
    >
      <span class="letter">{{ opt.key }}</span>
      <span class="text">{{ opt.text }}</span>
    </div>
    <button class="btn btn-primary btn-block" @click="submit">提交</button>
  </div>
</template>

13 · PRINCIPLES

设计原则

在动笔之前、之后都用这套原则自查。

坚持

  • 画布永远落在 --bg 雪色，绝不用纯白当唯一底色
  • 主操作只允许一种颜色（蓝天蓝），成就类用旭日，绝不混用
  • 状态色默认走「主色 + 柔色背景」组合，避免色块抢占注意力
  • 题面字号 ≥ 22px，按钮 ≥ 14px，正文对比度 ≥ 4.5:1
  • 阴影使用 OKLch + 5–10% 透明度，绝不使用 rgba(0,0,0,0.1)

避免

  • 紫色 / 紫罗兰渐变（典型 AI slop，破坏蓝天主轴）
  • 圆角 16px 以上的「胖卡片」配主色背景
  • 在答题过程中出现弹窗 / 全局通知打断节奏
  • 用 emoji 当 UI 图标（仅允许在教学题目内容里出现）
  • 冷热色混用（蓝色背景 + 红色按钮）
自检清单：① 一屏只一个主操作 ② 状态反馈用柔色背景 ③ 文字层级 ≤ 4 档 ④ 焦点环可见 ⑤ 阴影低饱和 ⑥ 答题过程无刺眼红/紫。
