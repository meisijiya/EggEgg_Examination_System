# Proposal: Finance Examination System (MVP)

## Why (问题背景)

女友备考财务管理科目期末考试。当前有大量零散资料（12 个 PDF：课件 / 课后习题 / 即测即评），需要：
- 系统性出题练习，模拟真实考试节奏
- 每道题明确章节归属，便于诊断弱项
- 错题有 AI 讲解辅助学习
- 数据可信：宁可少而准，不可多而错（女友成绩差 = 学习失败）

## What (目标产物)

部署在远程云服务器的 Web 考试系统，支持：
- 模拟考（41 题 / 100 分 / 120 分钟，贴近中级会计 / 财经类考试标准结构）
- 客观题自动判分 + 主观题关键词覆盖率判分
- 每题带章节归属 + 历次成绩趋势 + 章节雷达图
- 学员按需触发 AI 讲解（流式 SSE）
- 一份 `docker-compose up` 即可部署

## Goals (本期范围)

1. ✅ 财务管理科目（9 章 × 5 种题型）
2. ✅ 算法随机抽样出题（章节 × 题型 × 难度加权）
3. ✅ 关键词覆盖率判分（客观 + 主观统一规则）
4. ✅ 学员按需触发 AI 讲解（流式流）
5. ✅ 历史成绩 + 趋势 + 雷达图
6. ✅ Docker Compose 单容器部署

## Non-Goals (本期不做)

- ❌ 章节刷题模式 / 错题本 / 自由组卷
- ❌ 多用户 / 多租户 / 公开注册
- ❌ 移动 App / PWA / WebSocket 实时协作
- ❌ K8s / 微服务拆分
- ❌ RAG / 向量数据库
- ❌ LLM 出题改编（出题纯算法，LLM 仅讲解）
- ❌ 新增学科（架构允许，但本期不实现）

## 关键 Acceptance Scenarios

1. **Happy path**: 女友打开网页 → 登录 → 开始模拟考 → 41 题按章节覆盖答题 → 交卷 → 立即出分 + 章节分 → 点击错题 AI 讲解 → 流式显示讲解内容
2. **覆盖率**: 每次模拟考覆盖 ≥ 8/9 章，难度分布 ≈ 30% 易 + 50% 中 + 20% 难
3. **判分稳定**: 客观题 100% 准确；主观题关键词覆盖率 ≥ 60% 才给分
4. **AI 讲解失败回退**: 流中断 → 显示参考答案 + 解析（不影响考试本身）
5. **部署 1 命令**: `docker-compose up` 后浏览器访问 `https://<server>` 可登录

## Out of Scope (待 v2+)

- 新增学科（会计 / 审计）复用同一 pipeline
- 新增题型（填空 / 简答）
- 学员答案相似度检测（防搜索式答题）
- 移动端 PWA
